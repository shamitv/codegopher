# Contributing

CodeGopher v0.1 work should stay small, tested, and easy to review.

- Keep commits focused on one behavior, module, or test slice.
- Run the smallest relevant verification command before each commit.
- Push each completed implementation commit to the active upstream branch.
- Do not commit virtual environments, caches, generated build artifacts, or local session data.
- Prefer tests with the behavior they protect, especially for provider, tool, approval, and prior-read logic.

The current v0.1 implementation sequence lives in `docs/plans/initial/TODO.md`.
